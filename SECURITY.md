# Security Policy

## Supported Versions
We actively provide security patches for the following versions of CompliCore:

| Version | Supported | Notes |
| :--- | :--- | :--- |
| **main** | :white_check_mark: | Latest development builds. |
| **v1.0.x** | :white_check_mark: | Current stable release. |
| **< 1.0.0** | :x: | Legacy/Beta versions (no longer patched). |

## Reporting a Vulnerability
We take the security of CompliCore seriously. If you believe you have found a security vulnerability, please **do not** open a public issue.

### Preferred Method
Please submit a report via [GitHub Private Vulnerability Reporting](https://github.com/officechbusinessservices-creator/CompliCore/security/advisories/new). This allows us to collaborate with you privately to fix the issue.

### Alternative Method
If you cannot use GitHub's reporting tool, please email us at **security@complicore.com**.

## Our Response Process
* **Acknowledgement:** You will receive an initial response within **48 hours**.
* **Fix Timeline:** We aim to provide a resolution or mitigation plan within **30 days** for confirmed critical issues.
* **Public Disclosure:** We ask that you do not disclose the vulnerability publicly until we have released a fix. We will give you full credit for the discovery in our security advisories.

## Out of Scope
The following types of reports are generally considered out of scope:
* Social engineering or phishing attacks against maintainers.
* Denial of Service (DDoS) attacks.
* Vulnerabilities in third-party dependencies (these should be reported to the specific package maintainers).
